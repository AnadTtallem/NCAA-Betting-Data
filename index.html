Overview

This project aims to build a robust NCAA Betting System that leverages both historical data and live data to generate accurate betting predictions. The system is comprised of several integrated components:

  • Historical Data Ingestion: Loads and cleans historical game data (team stats, betting odds, outcomes) from CSV files.
  • Feature Engineering: Calculates features such as score margins, rolling averages, home/away indicators, and win streaks.
  • Machine Learning Module: Trains a predictive model (e.g., Gradient Boosting Classifier) using historical data.
  • Live Data Integration: Fetches real-time data (injuries, weather, live scores/rankings) via API endpoints.
  • Combined Prediction Service: Merges live and historical features to generate game outcome predictions.

Key Components & Steps

1. Historical Data Pipeline

Function: load_historical_data(zip_path)

Purpose: Reads a ZIP file containing CSV files from historical data, assigns a conference label based on the file name, and converts data types.

Usage:

  • Update the file path to your local copy of the historical data ZIP file.

Example:


zip_file_path = r"C:\Users\danam\ncaa-api\NCAA-Betting-Data-main.zip"
historical_df = load_historical_data(zip_file_path)

2. Feature Engineering Module

Function: add_features(df)

Purpose: Computes additional metrics (score margin, rolling averages, home game indicator, win streak) to enrich the data.

Usage:


historical_df = add_features(historical_df)

3. Machine Learning Model Training

Function: train_model(df)

Purpose: Uses the engineered features to train a Gradient Boosting Classifier to predict game outcomes.

Usage:


model = train_model(historical_df)

Evaluation: Prints cross-validated accuracy and test set accuracy.

4. Live Data Integration Module

Functions:

  • fetch_injuries_data()
  • fetch_weather_data(location)
  • fetch_live_rankings()

Purpose: Connects to external APIs to fetch real-time information on injuries, weather conditions, and rankings.

Usage:

5. Combined Prediction Service

Function: predict_game_outcome(model, live_features)

Purpose: Uses live data (transformed into features) to generate predictions via the trained model.

Usage:

    'rolling_margin': 4.5,
